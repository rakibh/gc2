6. 90-DAY AUTOMATIC RETENTION POLICY
================================================================

6.1 Policy Definition
---------------------
- Default retention period: 90 days
- Logs older than 90 days from current date/time shall be automatically deleted
- Retention period configurable via System Settings (key: log_retention_days)
- Minimum allowed retention: 30 days (safety threshold)

6.2 Cleanup Mechanism
---------------------
MySQL Event Scheduler (Recommended)
CREATE EVENT cleanup_system_logs
ON SCHEDULE EVERY 1 DAY
STARTS '2026-01-01 03:00:00'
DO
  DELETE FROM system_logs 
  WHERE created_at < NOW() - INTERVAL (SELECT setting_value FROM system_settings WHERE setting_key='log_retention_days') DAY;


6.3 Safety & Compliance
-----------------------
- Cleanup runs during off-peak hours (default: 3:00 AM)
- Batch deletion prevents long-running transactions
- Cleanup success/failure logged
- Optional archival step before deletion (if compliance requires >90 days)
- Admin can pause auto-cleanup via settings toggle

6.4 Configuration UI
--------------------
- Settings Page
- Field: "Log Retention Period (Days)" [Input: Number, Min: 30, Max: 365, Default: 90]
- Toggle: "Enable Auto-Cleanup" [Yes/No]
- Schedule: "Cleanup Time" [Time picker, Default: 03:00]
- STARTS date - dd/mm/yyyy

7. SECURITY & COMPLIANCE RULES
================================================================

SEC-LOG-001: All log entries are append-only (no UPDATE/DELETE via application)
SEC-LOG-002: Admin-only access for export, clear, and configuration
SEC-LOG-003: Sanitize all log inputs to prevent XSS/HTML injection
SEC-LOG-004: Mask sensitive data in context (passwords, tokens, PII)
SEC-LOG-005: Log all admin actions on the logs module itself
SEC-LOG-006: Rate-limit log writes to prevent log flooding attacks
SEC-LOG-007: Store IP addresses in IPv4/IPv6 compatible format (VARCHAR(45))
SEC-LOG-008: Comply with data minimization principles (no unnecessary PII logging)


8.3 Performance Optimization
----------------------------
- Batch inserts for high-frequency events (optional buffer)
- Index on created_at for fast range queries
- LIMIT/OFFSET pagination with index usage
- Avoid logging in tight loops or recursive functions
- Async logging via queue (future enhancement)

8.4 Integration Points
----------------------
- Auth Module: login, logout, failed attempts, password reset
- Equipment/Task/Network: CRUD actions, status changes, assignments
- Tools Module: backup, optimize, repair, settings changes
- Security Middleware: CSRF failures, permission denials, rate limits
- Error Handler: PHP exceptions, PDO errors, file system failures
